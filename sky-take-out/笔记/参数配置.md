在写阿里云的工具类时有一些工具类中要要配置参数

```java
@Component
public class AliyunOssOperator{
    private Stirng endpoint = "https://oss-cn-beijing.aliyuncs.com";
    private String bucketName = "java-ai";
    //其他函数
}

```

当这些容易变动的配置直接写在java代码中不易于项目维护
因此要将容易变动的参数定义在文件中
在springBoot中是定义在application.yaml中


- 参数配置化:将一些需要灵活变化的参数，配置在配置文件中，然后通过@Value注解来注入外部配置的属性

```yml
#application.yml
aliyun:
  oss:
    endpoint: https://oss-cn-beijing.aliyuncs.com
    buckName: java-ai
    region: cn-beijing


```


```java
@Component
public class AliyunOssOperator{
    
    @Value("${aliyun.oss.endpoint}")
    private String endpoint;
    @Value("${aliyun.oss.buckName}")
    private String bucketName;
    
}

```

缺点: 当注入的配置项较多，非常冗余

@ConfigurationProperties实现批量注入
将多个配置项的值全部封装入一个实体类中

```java

import jdk.jfr.DataAmount;

@Data
@Component
//这里的prefix="aliyun.oss"表示配置的是配置文件中前缀为aliyun.oss的值
@ConfigurationProperties(prefix="aliyun.oss")
public class AliyunOSSProperties{
    private String endpoint;
    private String bucketName;
    private String region;
}
```


```java
public class AliyunOSSOperator{
    @Autowired
    private ALiyunOSSProperties aliyunOSSProperties;
}



```
